DOM

  • ECMAScript : 标准语法

    • 进行一些数据的操作

  • DOM - document object model 文档对象模型

    • 它的出现就是通过document提供api（接口文档），赋予开发者操作页面的能力

    • 如果要操作某个元素，首先要先活期出来，document…..

    • 获取的元素是个对象，在js中，标签和对象是相映射的关系，修改对象下的属性有可能会改变标签，修改标签有可能会修改到对象身上

  • 难点 – 复杂关系

    • DOM树 - 把页面中的所有节点组织在一个树形结构中，方便我们通过关系去操作页面

    • 页面中的所有的内容都由不同节点组成（不同的节点组成一个页面）

      • 元素节点 -> 标签 1

      • 注释节点 -> 注释 8

      • 属性节点 -> 标签上的属性 2

      • 文本节点 -> 文字、回车、换行 3

      • 文档节点 -> document 9

    • 查看节点的类型：nodeType

    • 操作节点的内容： nodeValue textContent value

    • 读取节点的名字：nodeName

    • 属性节点： attributes

    • 查找当前元素下的所有子节点： childNodes

    • 找到某个元素下的元素子节点：children

      • 找到父级的第一个子元素：firstElementChild

      • 找到父级的最后一个子元素：lastElementChild

    • 下一个兄弟元素节点：nextElementSibibling

    • 上一个兄弟元素节点：previousElementSibling

    • 查找某个元素的父级节点：parentNode

    <script>

    /*  

     childNodes  获取某个元素下的所有子节点



     children   获取某个元素下的所有元素子节点



     需求：

         自己手写一个方法，这个方法返回某个元素下的所有元素子节点

         1.通过childNodes去拿到所有的子节点

         2.通过判断，这些子节点下哪个是元素子节点

         3.如果是就把这个元素子节点push到一个数组中

         4.最后把这个数组返回出去

     */

   function children(parent){ //函数定义

     // 我先看看这个parent到底是一个什么数据？ 如果是个字符串，那么我就获取元素。如果parent直接就是一个元素，就不需要获取了

//      if(typeof parent === 'string') {

//          parent = document.querySelector(parent);

//      }else if(parent.nodeType === 1) {

//          parent = parent;

//      }

   parent= typeof parent === 'string' ?  parent = document.querySelector(parent) : (parent.nodeType === 1 ? parent : null);



   // 如果为null，就说明参数是瞎传的

   if(parent === null) return '请正确传参';

   const childrens = parent.childNodes;

    let arr = [];

    for (let i = 0; i < childrens.length; i++) {

        if(childrens[i].nodeType === 1) {

            arr.push(childrens[i]);

        }



    }

   return arr;

   }

//    children('#ul'); // 函数调用

   console.log(children('#ul'));

    </script>

DOM增删改的方法

  • 创建元素

    • 创建元素 document.createElement(‘标签名’)

    • 让元素的末尾添加一个元素：parent.appendChild(child)

    • 删除元素 parent.removeChild(删除的元素)

回流：

        当某个元素发生了几何、坐标、位置...的变化，会使页面局部刷新，或者导致整个页面刷新



    重绘：

        元素几何、坐标.. 不发生变化，只是外观、颜色、风格发生变化的时候会导致整个页面重新绘制



       不管是回流还是重绘，都是性能的杀手，我们要尽可能少操作DOM，页面至少会回流一次，

       回流一定会重绘，而重绘不一定导致回流
